# Subcutaneous Test

## 要約

Subcutaneous Test は、UIそのものを通さず、UIのすぐ下にあるアプリケーションの機能を動かして検証するテストです。エンドツーエンドに近い振る舞いを確認したいが、UI経由の自動テストが難しい、遅い、壊れやすい場合に役立ちます。

## 読むときの観点

- UIを迂回することで、何を速く安定して確認できるのか。
- UIに重要なロジックが残っている場合、どんな振る舞いがテストから漏れるのか。
- UIテスト、統合テスト、機能テストの境界をどう引くか。

## 原文の翻訳

私が subcutaneous test という言葉で意味しているのは、アプリケーションのUIのすぐ下で動作するテストである。これは、アプリケーションの機能テストを行うときに特に価値がある。つまり、エンドツーエンドの振る舞いをテストしたいが、UIそのものを通してテストするのが難しい場合である。

Subcutaneous testing は、テストしにくいプレゼンテーション技術にまつわる困難を避けられるし、通常はUIを通したテストよりもずっと速い。大きな危険は、UIから有用なロジックをすべて追い出すという方針を厳密に守っていないかぎり、**subcutaneous testing が重要な振る舞いをテスト対象から外してしまう**ことである。
