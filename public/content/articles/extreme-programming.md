# Extreme Programming

## 要約

Extreme Programming（XP）は、Kent Beck を中心に発展したソフトウェア開発手法であり、初期のアジャイル手法の中でも大きな影響力を持っていた。Fowler は、XP がアジャイル手法への注目を集める主要なきっかけになり、アジャイル開発を始める土台としては Scrum より優れていると見ている。

XP は、価値、原則、具体的なプラクティスへと考えを進める構成を持ち、継続的インテグレーション、リファクタリング、テスト駆動開発、アジャイルな計画づくりなど、多くの実践を広めた。技術面とマネジメント面のプラクティスを組み合わせる点が、XP の重要な特徴である。

## 読むときの観点

- XP を、単なるプラクティス集ではなく、価値から具体的実践へ落とし込む考え方として読む。
- Scrum との違いを、管理プラクティスと技術プラクティスの組み合わせという観点で捉える。
- チームが「XP をやっているか」よりも、XP からどんな影響を受けて実践しているかに注目する。

## 原文の翻訳

Extreme Programming（XP）は、主に Kent Beck によって開発されたソフトウェア開発手法である。XP は最初期のアジャイル手法の一つであり、実際、2000年代が進むにつれて Scrum が主流になる前、1990年代後半から2000年代初頭にかけて支配的なアジャイル手法だった。多くの人々、私自身も含めて、XP はアジャイル手法に注目を集めた主要な触媒であり、アジャイル開発を始めるための土台としては Scrum より優れていると考えている。

Jim Shore による Agile Software Development の説明は、Extreme Programming の原則に大きく基づいており、XP のアプローチを学ぶための手引きとして私が最初に選ぶものだ。

Kent は、1980年代後半から1990年代前半にかけて Smalltalk プロジェクトのコンサルティングを行う中で XP を発展させた。のちに XP として知られるようになった一連のプラクティスは、C3 プロジェクトで初めてまとめて使われた。私はそのプロジェクトで Kent と働き、XP について学んだ。「Extreme Programming」という名前はその後、このアプローチが説明される中で生まれた。最初は WikiWikiWeb 上で非公式に説明され、のちに一連の書籍で説明されるようになった。

さまざまなチームが WikiWikiWeb 上の説明を取り入れて、自分たちで XP を実践した。これにより、この手法は再現され、もともとの場所以外でも使えることが示された。

Kent は XP を、広く抽象的な価値から原則へ、さらに具体的なプラクティスへと進む考え方の流れとして説明している。私はこの流れを、ほかの多くの文脈にも適用できる有用なものだと感じている。XP は、継続的インテグレーション、リファクタリング、テスト駆動開発、アジャイルな計画づくりなど、その後ソフトウェア開発で広く使われるようになった多くのプラクティスを普及させた。私が特に気に入っているのは、**技術プラクティスとマネジメントプラクティスを組み合わせている**点であり、それによって XP はアジャイル流暢性における delivering zone に到達するうえでよく合っている。

これは、意図的にマネジメントプラクティスだけを含める Scrum との対比である。そのため Scrum は、Flaccid Scrum になりやすい。

「white book」は Extreme Programming の決定的な説明であり、どのプログラマの読書リストにも入るに値する。

もっとも、たいていの XP 実践者と同じく、私もチームが XP をやっているかどうかで判断することに大きな意味があるとは思っていない。それでも、Thoughtworks のほとんどのプロジェクトは、主に XP の影響を受けたスタイルで進められていると言える。

### さらに読む

Extreme Programming の決定的な説明は、Kent の white book である。ただし注意してほしい。初期の XP 実践者の多くは第1版を使ってこのアプローチを学んでおり、第1版と現在の版との間にはかなり多くの違いがある。

White Book が XP の最良の定義である一方、今日では、XP のアプローチを学ぶための手引きとして James Shore の The Art of Agile Development を薦めている。Shore は Kent の XP に関する仕事よりも広い範囲を扱っているが、その本は XP の重要な特徴にしっかり根ざしている。
