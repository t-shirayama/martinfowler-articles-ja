# C3

## 要約

C3 は Chrysler Comprehensive Compensation project の略称で、Extreme Programming の「誕生プロジェクト」として知られるようになった給与計算プロジェクトです。既存の多数の COBOL 給与システムを置き換える試みとして始まり、1996年の再始動後に XP として知られる実践群がまとまりました。

この記事は、C3 の初期成功が XP に大きな影響を与えた一方で、後にプロジェクトが中止されたことは XP が成功を保証するものではないことも示している、と整理しています。Fowler 氏は、自身の関与が一部かつ1996年末までだったため、事実に基づく包括的な分析はフルタイムで関わった人から出るべきだと述べています。

## 読むときの観点

- C3 は XP の実践が最初に一つにまとまった歴史的プロジェクトとして読む。
- 初期成功とキャンセルの両方を、単純な成功談や失敗談にしない点に注目する。
- Fowler 氏自身が、プロジェクト全体の分析について慎重な距離を置いていることを確認する。
- Wikipedia などの二次情報に対する注意も、歴史的事例を読む姿勢として見る。

## 原文の翻訳

C3 は Chrysler Comprehensive Compensation project の略称で、Chrysler における給与計算プロジェクトでした。その後、[Extreme Programming](https://martinfowler.com/bliki/ExtremeProgramming.html) の「誕生プロジェクト」として有名になりました。

このプロジェクトは、存在していた多数のレガシーな COBOL 給与計算システムを置き換えようとする試みでした。私は1993年から、いくつかの形でコンサルタントとして関わりました。プロジェクトは1995年に Smalltalk で本格的な開発作業を始めましたが、安定した状態に到達できず、1996年に Kent Beck のリードのもとで再始動されました。**Extreme Programming として知られるようになったすべてのプラクティスを初めて一つにまとめたのは、この再始動後のプロジェクトでした**。ただし Kent は、それ以前のプロジェクトでも似たアプローチを使っていました。

ソフトウェアは1997年に本番稼働し、およそ1万人に給与を支払いました。プロジェクトはその後も、より大きな範囲の給与計算を担うことを意図して続きましたが、1999年に新規開発が止まりました。その後、稼働中だった既存システムは COBOL に戻されました。ただし、私が最後に聞いたところでは、既存ソフトウェアでは扱えない少数の人たちには、まだそのシステムが給与を支払っていたようです。また、将来は ERP システムを使って給与計算を扱う意向だとも聞きました。

C3 の初期成功は XP にとって大きな刺激となり、C3 は XP の道を切り開くうえで重要な役割を果たしました。プロジェクトがキャンセルされた頃には、C3 の初期成功を再現したほかのプロジェクトが十分にあり、XP の発展は続くことができました。しかし、C3 のキャンセルは、**XP が成功を保証するものではない**ことも示しています。

多くの人が、C3 の成功とキャンセルを分析しようとしてきました。私は、事実について十分な知識に基づいた分析を見たことがありません。完全な説明は、本当はチームにフルタイムでいた人たちから出るべきだと、私はずっと感じてきました。そのため、私自身は何も書いてきませんでした。私はパートタイムで関わっただけで、しかも1996年の終わりまででした。特に Wikipedia の項目は誤解を招き、不完全です。そのコメントの多くは、出典が不明確な、断固とした XP 批判者の論文に基づいているように見えます。

性能に関するそのコメントは、私の『Refactoring』本にある素材を誤って解釈したものだと言ってよいでしょう。

C3 チームの多くは、その後 Chrysler の内外で、さらに XP の仕事へ進みました。あるグループは、[VeryLowDefectProject](https://martinfowler.com/bliki/VeryLowDefectProject.html) として注目されるプロジェクトで、今も XP を広く使い続けています。
