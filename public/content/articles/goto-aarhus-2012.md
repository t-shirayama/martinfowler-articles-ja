# goto Aarhus 2012

## 要約

Martin Fowler が GOTO Aarhus 2012 を振り返り、当時の技術潮流として NoSQL を中心とするデータ技術、ブラウザ上の JavaScript、Microsoft のオープンソースへの姿勢の変化、そして SVG がもたらす表現力を取り上げています。単なるイベント報告ではなく、開発者が新しいデータの世界やブラウザ技術をどう受け止めるべきかを考える短い所感です。

## 読むときの観点

- NoSQL や新しいデータ技術が、開発者の役割を受動的な「要求の実装者」から押し戻す可能性に注目する。
- JavaScript の欠点を認めつつ、それでも第一級の言語として扱わざるを得ない状況を読む。
- Microsoft とオープンソースの関係について、当時の空気感を踏まえて読む。
- SVG を単なる画像形式ではなく、DOM と結びついたインタラクティブな表現基盤として捉える。

## 原文の翻訳

この数か月は移動が多かった。終盤に数えてみると、直近44日のうち40日を旅先で過ごしていた。そのため、このサイトもしばらく静かになっていた。ようやく家に戻ってきたので、その間のことを少し振り返れる。GOTO conference は、いつも考える材料に満ちた場だ。

GOTO / InfoQ のカンファレンスは、データに関する新しい考え方が流行になる前から、そこに大きな重点を置いてきた。今年もその話題は多かった。NoSQL 技術に強く焦点を当てたトラックがいくつかあり、一方では主要な NoSQL データベースをすばやく概観し、もう一方ではそれらの応用に集中していた。

Adrian Cockcroft は、Netflix がコンピュータインフラストラクチャを構築するなかで学んだ多くの教訓を共有した。Netflix による Cassandra の大規模利用は特に興味深かったし、ここで少し時間を取って、彼らが学びやツールをオープンに共有していることに敬意を表したい。私たちの職業の進歩は、実際の競争優位というより虚栄から生まれる秘密主義によって妨げられることがあまりに多い。Netflix は、境界を押し広げながら、その過程でツールを共有していく企業のすばらしい例だ。

実のところ、**新しいデータの風景**は、GOTO だけでなく、この旅全体を通じたテーマでもあった。私の同僚の多くも、クライアントワークと余暇の両方で、その可能性を探っている。いま私がコンピューティングの中で最も強く引きつけられている領域であり、新しい技法を探り、利用者の仕事をソフトウェアでより良くする方法を見つける機会がたくさんある。

これはまた、開発者を要求の受け身の受領者と見なす残念な傾向に対抗する機会でもある。新しいデータの世界がもたらす多くの機会は、技術に近い人でなければ見えない。だから私は、これが私たちを Conversational Stories の方向へ少し引き戻してくれることを期待している。

もう一つ印象に残ったテーマは、JavaScript の中心性が高まっていることだった。Scott Hanselman の基調講演では、ブラウザで動く JavaScript が、多くの人にとって意外に思えることを実現している様子が披露された。そしてもちろん、Anders Hejlsberg の締めくくりの基調講演もあり、JavaScript の途方もない複雑さを手なずける別のアプローチが示された。

Clojure、F#、Scala のような現代的で洗練された言語に深く入り込んでいる同僚たちに、私はよく「JavaScript こそ未来の唯一の言語だ」と冗談を言う。人をからかうのは楽しいが、愉快な考えではない。私は、これほど深い欠点を持つ言語がコンピューティングでこれほど重要な役割を果たしていることに絶望する側の人間だからだ。

それでも、その帰結として、私たちは **JavaScript を第一級の言語として真剣に扱い**、その欠点が引き起こす損害をどう抑えるかに集中しなければならない。Microsoft の TypeScript のアプローチには好ましい点が多く、私自身も今後数か月で使ってみることになりそうだ。正気の人間なら望まないほど多くの JavaScript を含む仕事に、もう少し時間を使えることを期待している。

JavaScript は Scott と Anders の講演に共通するテーマだったが、同じくらい重要な要素がもう一つあった。私は2007年に、Microsoft がオープンソースの世界と共存するのに苦労していることについて書いた。その後、その短い記事が小さな騒動を起こし、Microsoft の何人かの管理職が私の批判を非難し、私を救いようのない敵だと決めつけたことを知った。そうした反応は気落ちするものだったが、幸い、もっと強い力も働いていた。

ASP.NET / Azure の世界にいる Scott の同僚たちも、TypeScript チームも、数年前なら衝撃的に見えたであろう程度のオープンさを持っている。TypeScript は CodePlex 上にあり、自分自身で実装されているため、Windows と同じくらい容易に Mac やオープンソースの Unix 系環境でも動かせる。

私の感覚では、Microsoft は非常に派閥的な組織だ。オープンソースの世界にいまだ強く敵対的な派閥も多いだろう。しかし、オープンソースとの共存に取り組む大きな派閥がこれほど存在するなら、長く「悪の帝国」への不信を抱いてきた多くの開発者も、疑いの向け先を考え直してよいはずだ。

Anders Hejlsberg の基調講演は、GOTO の基調講演で、JavaScript の欠点を修正する重要技術が語られた二年連続の例でもあった。

ブラウザ中心の未来について語ったもう一人の講演者が Dan North だ。彼は、ブラウザアプリケーションの進化について、満員の部屋で実に楽しい講演をした。その中で彼は、グラフィックスに関する多くの発展にどれほど関心がないかを示していた。私もその無頓着さにはかなり共感するが、一つだけ意見が違う技術がある。SVG だ。

SVG は、Web 上でベクターグラフィックスを扱うための標準である以上のものだ。より重要なのは、SVG が **ベクター形状を DOM の要素として作る**ことであり、それらは JavaScript や CSS からアクセスできる。この点は、私には深い変化に思える。その帰結の一つは d3 gallery を眺めれば分かる。そこでは、データ可視化の世界がどのように捉え直されているかを見ることができる。現時点の Web はまだ静的なテキストと画像の世界であり、動画は存在するものの、基本的には異物のようなものだ。

SVG は、アニメーションとインタラクティビティのための道具箱を開く。これは、アプリケーション UI と出版物の両方において、ブラウザの使い方に大きな違いをもたらすかもしれないと私は考えている。
