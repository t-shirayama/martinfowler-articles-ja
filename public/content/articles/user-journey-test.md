# User Journey Test

## 要約

User Journey Test は、典型的な利用者がシステム上で目的を達成するまでの一連の流れをなぞる、ビジネス向けのテストです。システム全体の統合を確認するのに役立つ一方、広い範囲を通るため遅く壊れやすくなりがちです。

そのため、あらゆる振る舞いを網羅するためではなく、主要なユースケースごとに代表的な経路を少数だけ置くのが基本です。細かな分岐や例外条件は、より焦点を絞った別種のテストで確認します。

## 読むときの観点

- ユーザーの目的達成までをひとつの経路として捉える
- システム全体の統合確認と、詳細な振る舞いの網羅を分けて考える
- Story Test との違いを、ユーザーストーリー単位かユーザー経路単位かで見る

## 原文の翻訳

User Journey Test は Business Facing Test の一種であり、典型的なユーザーがシステムの中を進む「旅路」をシミュレートするためのものです。この種のテストはふつう、ユーザーがある目標を達成するまでの、システムとのやり取り全体を扱います。ユースケースの中のひとつの経路を表すものだと言えます。

User Journey Test は、多くの場合 Broad Stack Test でもあります。そのため、実行に時間がかかり、壊れやすくなりがちです。したがって、User Journey Test のスイートは通常、システムの振る舞いを網羅的に検証するためには作られません。むしろ、システム全体の統合を動かして確認するために、少数の User Journey Test を置くのが普通です。おそらく各ユースケースにつきひとつ、たいていは happy path を確認する程度です。

振る舞いのすべてのバリエーションを検証する仕事は、別のスタイルのテストに任せます。通常は、より焦点を絞ったカバレッジを持つテストを使います。**User Journey Test は網羅のためではなく、全体のつながりを代表的な経路で確認するためのテスト**です。

Story Test と対照的に、User Journey Test はユーザーストーリーに直接結びついているわけではありません。あるストーリーに取り組むときは、既存の User Journey Test を見て、そのユーザーストーリーが示す振る舞いの変更を支えられるように修正します。ユーザーストーリーがまったく新しい User Journey Test を生むことは、まれです。
