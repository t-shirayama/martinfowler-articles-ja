# Pervasive Versioning

## 要約

Apple の Time Machine は、自分のファイルの変更履歴をあとからたどれるようにする機能です。これはバックアップとして見られがちですが、バージョン管理の考え方をより多くの人とアプリケーションに広げるきっかけにもなります。

## 読むときの観点

- バージョン管理をソフトウェア開発者だけの道具ではなく、日常的な作業全体に広げる発想として読む。
- 自動バックアップと、意図を持ってコミットするバージョン管理の違いに注意する。
- 差分とマージをアプリケーションが理解できるようになることの重要性を見る。

## 原文の翻訳

Apple は最近 Time Machine を発表しました。これは、時間をさかのぼってファイルに加えられたすべての変更を見たり、削除されたファイルを見つけたりできる機能です。熱心な技術好きの一部にとって、これは新しい機能ではありません。私も他の人たちと同じように、自分の作業ディレクトリ全体をバージョン管理下に置いています。最初は CVS、今は Subversion を使っています。そのおかげで、自分が扱うものすべてについて、変更履歴を簡単に見られるようになっています。これはとても便利なので、以前から「もっと多くのものがバージョン管理される」とどうなるだろうと考えてきました。Time Machine は、その方向への一歩かもしれません。

Time Machine は自動バックアップシステムとして見られています。そのため、バージョン管理システムにあるような、考えて行うコミットという概念は支えていないようです。少なくとも最初の段階では、これはよい進め方だと思います。人々がこうした種類のシステムに慣れるためです。時間を軸にしたブラウザは面白そうです。バージョン管理システムにはユーザーインターフェイスの再考が必要であり、それをやるのに Apple ほどふさわしい存在もなかなかありません。

より重要なのは、この能力が広く使えるようになることで、アプリケーション開発者への刺激になることです。以前「More Version Control」で、十分な数のアプリケーションが差分とマージを理解していないと書きました。Time Machine が、人々にそのことを考えさせ、こうした能力をアプリケーションに組み込むきっかけになるかもしれません。そうなれば、バージョン管理はずっと使いやすくなります。

バージョン管理は一台のデスクトップ上でも便利ですが、使ったことがある人なら誰でも知っているように、**本当の利点は共同作業にあります**。ソフトウェアプロジェクトでは、共同作業の道具としてバージョン管理システムを使うことで大きな利益を得ています。他の作業にも同じことが言えます。プレゼンテーション、ホワイトペーパー、Excel モデルも、バージョン管理された共同作業ツールから恩恵を受けられます。ここでも、賢い差分とマージが不足していることが大きな障壁です。私のように一人で作業する場合でさえ、複数のデスクトップで作業するときには大きな助けになります。

したがって私の望みは、Time Machine が、バージョン管理を意識し、それを活用できるアプリケーションの開発を促すことです。それがさらに、より効果的な共同作業へとつながっていくでしょう。いずれにしても、今すぐ試してみることを強く勧めます。Subversion は無料で、設定も簡単です。アプリケーションが差分とマージをうまく扱えないとしても、共有されたバージョン管理リポジトリを使って他の人と協力することには十分な価値があります。メールで送られてきた文書を追いかけたり、バージョン管理されていない共有ドライブを使ったりするより、ずっとよい方法です。
