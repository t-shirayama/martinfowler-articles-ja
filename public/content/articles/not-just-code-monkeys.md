# Not Just Code Monkeys

## 要約

この動画は、Fowler がアジャイルソフトウェア開発に対して抱く大きな問題意識から始まります。それは、ユーザー、アナリスト、プログラマの相互作用のあり方です。

講演は、これらの役割をたどりながら、プログラマとユーザーの関係、ユーザーに対する責任、そしてプログラマが向き合うべき「二つの大きな課題」へ進みます。公開ページ上で取得できたのは、動画そのものの完全な字幕ではなく、martinfowler.com と InfoQ に掲載された説明文です。

## 読むときの観点

- プログラマを「コードを書く人」だけに閉じ込める見方への違和感に注目する。
- ユーザー、アナリスト、プログラマの関係が、ソフトウェアの価値と責任にどう影響するかを考える。
- 技術者の責任を、実装範囲だけでなく社会的影響まで広げて読んでみる。

## 原文の翻訳

これは説明するのが難しい講演だ。ふつう私は、講演の内容を表すタイトルと要旨を好む。しかしこの講演は一つの旅であり、どこへ向かうのかを先に言いたくない。むしろ、私と一緒にその地面を探っていきたいのだ。

ただし、出発点だけは言っておこう。この講演は、私がアジャイルソフトウェア開発の多くの導入について抱いている最大の問題、つまりユーザー、アナリスト、プログラマの相互作用の性質から始まる。

そこから、これらの役割を探っていく。プログラマとユーザーの関係、ユーザーに対する私たちの責任、そして最後に、プログラマが向き合う必要があると私が考える「二つの大きな課題」について問いを投げかける。

InfoQ の公開ページでは、この講演は「ソフトウェア開発が栄えるための健全な社会的環境を築くことの重要性」についての Martin Fowler による基調講演として紹介されている。また、Martin Fowler はソフトウェア開発に関する著者、講演者、コンサルタントであり、ThoughtWorks の Chief Scientist として紹介されている。
