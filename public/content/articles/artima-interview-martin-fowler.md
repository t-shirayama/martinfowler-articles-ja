# Interview with Artima.com

## 要約

Artima.com の Bill Venners 氏による Martin Fowler 氏への6部構成のインタビュー一覧ページです。
一覧では、リファクタリング、設計原則、進化的設計、柔軟性と複雑さ、テスト駆動開発、性能調整とプロセスが取り上げられています。

このページでは、Artima のインタビュー目次ページに掲載されている公開テキストを翻訳します。
各部の詳細本文は、この目次ページからリンクされている別ページとして扱います。

## 読むときの観点

- リファクタリングと設計を別々の活動ではなく、相互に支え合うものとして見る。
- 柔軟性、再利用性、単純さのトレードオフに注目する。
- テストファースト設計と、単体テスト・機能テストの違いを見る。

## 原文の翻訳

### Martin Fowler との対話

Bill Venners によるインタビュー、2002年12月9日。

この6部構成のインタビューで、Thoughtworks, Inc. の chief scientist であり、ソフトウェア設計とプロセスに関する多数の書籍の著者である Martin Fowler 氏は、リファクタリング、設計、テスト、Extreme Programming について語っています。

### Part I: Refactoring with Martin Fowler

Martin Fowler 氏が、リファクタリングとテストのビジネス上の理由、そしてリファクタリング、設計、信頼性の相互作用について語ります。

### Part II: Design Principles and Code Ownership

Martin Fowler 氏が、重複を避けること、プレゼンテーションロジックとドメインロジックを分離すること、明示的にすることといった設計原則について語り、リファクタリングがコード所有権にどう依存するかを説明します。

### Part III: Evolutionary Design

Martin Fowler 氏が、計画的な設計と進化的な設計、コードに現れる bad smells のリファクタリング、そして良い設計がプログラムをより速く書く助けになる理由について語ります。

### Part IV: Flexibility and Complexity

Martin Fowler 氏が、設計の劣化とどう戦うか、柔軟性と再利用性のコスト、単純なシステムの四つの基準、そしてインターフェースのテストファースト設計について語ります。

### Part V: Test-Driven Development

Martin Fowler 氏が、テストファースト設計と monological thinking の落ち着いた性質、そして単体テストと機能テストの違いについて語ります。

### Part VI: Tuning Performance and Process

Martin Fowler 氏が、保守性と効率のバランス、調整可能なソフトウェアを作ること、パターンの役割、そして Agile Software Manifesto について語ります。

### Martin Fowler 氏サイトでの紹介

私は、Bill Venners 氏から彼の Artima.com Web サイトのためにインタビューを受けました。
そのインタビューでは、リファクタリング、進化的設計、柔軟性、テスト駆動開発に触れています。
Bill は、私が筋の通ったことを話しているようにさえ見せてくれています。
