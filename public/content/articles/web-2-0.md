# Web2.0

## 要約

Web 2.0 は、単なる流行語ではなく、Web の性質に起きていた実際の変化を指す言葉として説明されています。Web をプラットフォームとして扱い、集合知、データ、継続的なリリース、軽量なプログラミングモデル、複数デバイス、豊かなユーザー体験といった観点から、その変化を理解しようとする議論です。

## 読むときの観点

- Web 2.0 を「新技術の一覧」ではなく、Web の使われ方の変化として読む。
- Tim O'Reilly の7つの原則が、処方箋ではなく観察から導かれた記述である点を見る。
- 言葉が広まるほど意味が薄まる Semantic Diffusion の例としても捉える。

## 原文の翻訳

ここ数年、Web 2.0 について多くの議論がありました。その概念についても、造語としての価値についてもです。私自身の関わりは限られています。Tim O'Reilly がこの話題について書いたり話したりしているものを読み聞きし、彼が企画したワークショップに参加した程度です。それでも、この言葉にはかなりの混乱があるので、その混乱を減らすために、うまくいかないかもしれない試みをしてみます。ここではかなりの部分で Tim の考えを私が解釈しているので、もし私と Tim の間に違いがあるなら、Tim のほうを信じてください。

Web 2.0 をめぐる混乱の多くは、それが何であるかをよく知らないまま語られていることから来ています。この言葉は、2004年のカンファレンス準備のブレインストーミングから生まれました。Web は崩壊したどころか、以前にも増して重要になっており、新しいアプリケーションやサイトが次々に現れている。そしてドットコム崩壊を生き残った企業には、いくつかの共通点がある。そうした認識から Web 2.0 という概念が出てきました。

この背景はとても示唆的です。Web 2.0 という言葉は、その当時、そして現在も進行していた World Wide Web の性質の変化を指しています。**この変化こそが、Web 2.0 の意味の核心**です。

専門家ではない普通の人にとっては、Web 2.0 はインターネット上で何か新しいことが起きている、という意味だと捉えられます。MySpace、YouTube、Flickr のような人気サイトや、ブログの広がりがそれに含まれます。人々がオンラインで何かを作り、共有するようになったという傾向であり、現象です。

Tim は、単にその傾向に名前をつけただけではありません。彼の Web 2.0 に関する議論の多くは、その傾向を理解しようとするものでした。特に、新しい Web の世界で成功しそうなものは何かに焦点を当てています。彼の定義的な記事では、Web 2.0 の7つの原則が挙げられています。

まず、Web をプラットフォームとして扱うことです。Netscape のようにデスクトップ上で動くソフトウェアではなく、Google のように Web 上のサービスとしてソフトウェアが動くという考え方です。

次に、集合知を活用することです。少数の専門家に頼るのではなく、多くの人からの情報を集めます。Amazon のレビュアーコミュニティや、リンクを検索アルゴリズムに使う Google はその例です。

3つ目は、データが次の「Intel Inside」になるという考え方です。Amazon の商品データベースや NavTeq の地図データのように、専門化されたデータベースを所有することが価値を持ちます。

4つ目は、ソフトウェアリリースサイクルの終わりです。新しい機能を高い頻度でリリースします。Flickr が非常に短い間隔で新しいビルドを配備していたことは、その例として挙げられます。

5つ目は、軽量なプログラミングモデルです。利用者が再利用しやすい、単純な開発環境やインターフェースを提供します。Google Maps の単純な連携インターフェースがこの方向にあります。

6つ目は、単一のデバイスを越えるソフトウェアです。Web と連携しながら、多くのデバイスが一緒に働きます。iTunes と iPod の組み合わせがその例です。

7つ目は、豊かなユーザー体験です。より動的なユーザーインターフェースによって、Web とリッチクライアントの差が縮まります。

これらの原則は、処方的というより記述的なものだと理解することが重要です。Tim は、何をすれば Web 2.0 になるかという指示を出しているのではなく、成功した Web サイトに何があったかを振り返っています。また、これらは Web 2.0 が何を意味するかについて、かなり一貫した説明にもなっています。Web 2.0 は無意味な言葉だという人に出会うこともありますが、そういうとき私は、Tim の論文を読んだのだろうかと思ってしまいます。

一方で、専門家が共有できる明確な理解を持てないなら、それはよい専門用語ではないという Nicholas Carr の見方にも、ある程度の共感はあります。ただし、その明確さの欠如は Semantic Diffusion、つまり意味の拡散という避けがたい過程によるものであり、完全でも止められないものでもありません。

よく出会う誤解のひとつは、Web 2.0 がすべて新しいものだという考えです。Web 2.0 に含まれる考えの多くは実際には古く、Web の初期から使われてきました。Amazon のコミュニティや Wiki の利用などがそうです。重要なのは、当時それらが少数派だったという点です。Web 2.0 の変化とは、1990年代には一部の人が使っていた原則が、将来には多数派になっていくことなのです。

全体として、Web 2.0 という言葉が指している根底の変化は、重要で現実のものだと思います。したがって、Web 2.0 という概念は妥当です。この概念に Web 2.0 という名前をつけるのがよいかどうかは議論できるでしょう。しかし、用語がすべての人を満足させることはありませんし、この言葉は定着してしまったので、いまさらそこを議論しても得るものは少ないでしょう。

より面白い議論は、原則についてです。**この現象を説明する原則として、それらは適切な組み合わせなのか**。そこからさらに、これらの原則と調和するパターンやプラクティスについて、より具体的に話すことができます。Tim が始めたかったのはその議論であり、私が関心を持っているのもそこです。
