# Ui Patterns Readings

## 要約

Ui Patterns Readingsは、Martin FowlerがUI patternsに関連して気に入った文章へのリンクを控えておくための短いメモです。2006年の夏にUI patternsの作業をかなり進めたものの、その後の執筆の中心はDomainSpecificLanguagesへ移ったと述べています。

ここでは、Jeremy MillerによるC#でのUI設計に関する連載と、Derek GreerによるMVC/MVPアーキテクチャ史の概観が紹介されています。

## 読むときの観点

- この記事自体は解説記事というより、関連文献メモとして読む。
- UI patternsの作業が一時停止している背景を押さえる。
- MVC/MVPなど、プレゼンテーションロジックの構造に関する読書案内として見る。

## 原文の翻訳

2006年の夏、私はUI patternsについてかなり大きな作業をしました。それ以来、それらはほとんど凍結状態にあります。私の主な執筆の焦点が、あまり目に見える形ではありませんが、DomainSpecificLanguagesへ移ったためです。このページには、その作業に関連して私が気に入った文章へのリンクを控えておきます。

ここ数週間、Jeremy MillerがC#でのUI設計について、素晴らしい連載記事を書いています。

Derek Greerには、MVC/MVPアーキテクチャの歴史についての有用な概観があります。
