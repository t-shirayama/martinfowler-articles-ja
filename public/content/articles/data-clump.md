# Data Clump

## 要約

Data Clump は、同じデータの組み合わせがあちこちに現れるコードスメルです。まとまりとして意味を持つデータをオブジェクトや値にまとめることで、意図を明確にできます。

## 読むときの観点

- いつも一緒に渡される引数やフィールドに注目する。
- データのまとまりが、隠れた概念を表していないかを見る。
- まとめることで振る舞いも移せるかを考える。

## 原文の翻訳

Data Clump は、複数のデータ項目がいつも一緒に現れる状態です。同じ引数の組がいくつものメソッドに渡されていたり、同じフィールドの組が複数のクラスに出てきたりする場合、それらは一つの概念を表しているかもしれません。

たとえば開始日と終了日が常に一緒に使われるなら、それは期間という概念かもしれません。住所の複数の項目がいつも一緒なら、住所オブジェクトとして扱えるかもしれません。

Data Clump の問題は、概念がコードに現れていないことです。項目がばらばらに扱われると、関係が読み取りにくく、変更時にも漏れが起きやすくなります。

改善の方向は、**一緒に動くデータを名前のあるまとまりにする**ことです。クラスや値オブジェクトを導入すると、データの関係が明確になります。さらに、そのデータに関わる振る舞いも移せるなら、より豊かなモデルになります。

Data Clump は単なる引数の多さではありません。いつも一緒に現れることが手がかりです。その繰り返しの背後にある概念を見つけることが、リファクタリングの出発点になります。
