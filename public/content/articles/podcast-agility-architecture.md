# Agile Architecture with Martin Fowler and Rebecca Wirfs-Brock

## 要約

Agile Uprising Podcast のエピソード紹介ページです。Martin Fowler 氏と Rebecca Wirfs-Brock 氏が、アジャイルにおけるアーキテクチャの意味、文脈によって変わるアーキテクチャの定義、単体テストや創発と事前設計の関係、ドメインモデル、ドキュメントについて話しています。

このページでは、公開されているエピソード紹介文を日本語に訳します。会話全体の transcript は公開ページ上で確認できないため、音声内容を推測して補ってはいません。

## 読むときの観点

- アーキテクチャを固定的な定義ではなく、製品や開発文脈に応じて変わる判断として見る。
- 単体テスト、創発的設計、事前設計がどのようにアーキテクチャに影響するかに注目する。
- ドメインモデルの定義に、ビジネス側の人々が関わる必要があるという点を見る。
- アジャイルにおけるドキュメントの立ち位置を、「コードがドキュメントである」という考えだけで片づけない。

## 原文の翻訳

### Agile Architecture with Martin Fowler and Rebecca Wirfs-Brock

2017年5月13日。

Agile Architecture を扱うこのエピソードを共有できることを、私たちはとても嬉しく思います。Martin Fowler が再び登場し、Rebecca Wirfs-Brock は初めて出演します。Martin の最初のインタビューは Agile Manifesto Author プロジェクトの一部として公開されており、そちらで聞くことができます。

Martin は Uprising podcast の友人であり、彼についての詳しい情報は、以前の投稿にあります。Rebecca は、Martin 自身が自分のアジャイルな考え方を研ぎ澄ます助けになった人であり、刺激を受けている相手だと述べている人物です。彼女は、オブジェクト設計に対する最初の振る舞い中心のアプローチである **Responsibility-Driven Design** の考案者です。また、1992年の Smalltalk Report の記事で object role stereotypes について書いており、これは UML における stereotype の概念にも影響を与えました。

彼女が考案した対話形式、つまり2列形式の use case は、その後 Larry Constantine によって広められました。最近の「driven」と呼ばれる設計アプローチの多くは、RDD にルーツがあり、RDD から影響を受けていることを認めています。

このエピソードの中で、Martin は IEEE Software magazine に書いた記事にすぐ言及します。その記事はリンク先で読むことができます。Martin と Rebecca は、会話の出発点として、アーキテクチャとは何かについて非常に明確な定義を示します。そこから、製品固有の文脈の中でアーキテクチャがどのように定義されるのかについて、率直な会話へ進んでいきます。また、何がアーキテクチャと見なされ、何が見なされないのか、その力学についても優れた洞察を示し、**定義はエンジニアリングの文脈に応じて流動的である**ことを説明します。

私たちは、単体テストがアーキテクチャに与える影響、そしてテストや創発がどの程度アーキテクチャを定義し、事前設計がどの程度それを定義するのかについても議論します。

また、ドメインモデルの重要性と、誰がドメインモデルの定義に関わるべきかについても話します。特に、**ビジネス側の人々がその会話に入る必要がある**という点を取り上げます。

インタビューの終盤では、アジャイルアーキテクチャにおけるドキュメントの重要性について議論します。議論は、「コードこそがドキュメントである」という立場から、より包括的なドキュメントの立場までを扱います。

このインタビューは2017年5月に行われ、Ryan Lockard がホストを務めました。
