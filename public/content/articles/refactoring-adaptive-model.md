# Refactoring to an Adaptive Model

## 要約

変化しやすいルールや構造を、より適応しやすいモデルへ移すリファクタリングを扱います。固定的な分岐や散らばった知識を整理し、変更が起きる場所を限定する考え方を示します。

## 読むときの観点

- 変化する知識がコードのどこに散らばっているかを見る。
- 適応的なモデルが柔軟性と複雑さの両方を持つ点に注意する。
- 抽象化を先取りせず、変化の圧力からモデルを導く姿勢を読む。

## 原文の翻訳

多くのシステムでは、ビジネスルールや分類、振る舞いの選択が時間とともに変化します。最初は単純な条件分岐で十分でも、変更が増えるにつれて、同じ知識が複数の場所に散らばり始めます。

そのようなとき、コードを適応的なモデルへリファクタリングすることがあります。目的は、変更されやすい知識を一か所に集め、ルールの追加や変更をより安全にすることです。

ただし、適応的なモデルは万能ではありません。柔軟な構造は、それ自体が複雑さを持ちます。したがって、まだ変化の方向が見えていない段階で先回りして導入すると、理解しにくい仕組みだけが残ることがあります。

大切なのは、**実際の変更の圧力から抽象化を引き出す**ことです。複数の変更が同じ場所を揺さぶり、似た修正が繰り返されるなら、そこにはモデル化すべき概念があるかもしれません。

リファクタリングによって適応的なモデルへ進むときも、段階的に進めます。既存の振る舞いを保ち、テストで支えながら、ルールの表現を少しずつ移していきます。その結果、変化に合わせて育つモデルを得られます。
