# The Manifesto for Agile Software Development

## 要約

この初期の記事は、2001年2月にユタ州 Snowbird で開かれた会合から Agile Manifesto が生まれた背景を説明します。集まったのは、Extreme Programming、Scrum、DSDM、Adaptive Software Development、Crystal、Feature-Driven Development、Pragmatic Programming など、当時「軽量」な開発方法と呼ばれていた流れに関わる17人でした。

記事の中心にあるのは、アジャイルは方法論に反対する運動ではない、という説明です。むしろ、方法論という言葉の信頼を取り戻し、モデリング、文書、計画を現実に役立つ範囲へ戻すことが狙いでした。価値あるものを時間どおりに届けるために、人、協働、信頼、文化を重視する姿勢が強調されています。

## 読むときの観点

- Agile Manifesto を、特定のプロセスの発明ではなく、複数の実践者が見つけた共通項として読む。
- 「軽量」という呼び名への違和感から、「agile」という言葉が選ばれていく流れを見る。
- アジャイルを反文書・反計画ではなく、文書や計画の目的を問い直す運動として読む。
- 技法の一覧よりも、信頼、尊重、協働に基づく組織文化への問題提起として読む。

## 原文の翻訳

2001年2月11日から13日にかけて、ユタ州 Wasatch 山脈の Snowbird スキーリゾートにある The Lodge に17人が集まりました。目的は、話し合い、スキーをし、くつろぎ、共通の土台を探ることでした。そこから生まれたのが、Agile Software Development Manifesto です。

会合には、Extreme Programming、Scrum、DSDM、Adaptive Software Development、Crystal、Feature-Driven Development、Pragmatic Programming などの代表者や、文書駆動で重厚なソフトウェア開発プロセスに代わるものが必要だと考える人たちが集まりました。

参加者は、それぞれ独立した考えを持つ実践者であり、ときには互いに競合する立場にもありました。それでも彼らは Agile Alliance と名乗り、ソフトウェア開発のための宣言に合意しました。ここで重要なのは、宣言が単なる手法のリストではなく、**互いへの信頼と尊重**に基づく価値観を示したことです。

記事は、アジャイルな方法論の中心には「人を最も重要な資産として扱う」と言うだけでなく、実際にそう行動する環境があると述べます。関心が高まった理由も、批判を受けた理由も、最終的には価値と文化に関わる部分にあります。

Extreme Programming が注目されたのも、ペアプログラミングやリファクタリングという個別の実践だけが理由ではありません。全体として見ると、それらの実践は、硬直した企業的手続きの重荷から開発者の共同体を解放するものとして働いたからです。

記事では Kent Beck の経験も紹介されます。彼はかつて、2人なら6週間で終えられる作業だと見積もりました。しかし開始時にもう1人のプログラマが外され、結果として12週間かかりました。後になって彼は、自分の見積もりが失敗だったのではなく、2人分の仕事を1人に押しつけた固定的なプロセス思考の失敗だったと気づきます。

このような問題は、ソフトウェア開発だけに限りません。マーケティング、管理職、外部顧客、内部顧客、そして開発者自身も、難しいトレードオフを避けるために、組織上の力関係によって不合理な要求を押しつけてしまうことがあります。

アジャイルなアプローチが官僚的な組織を不安にさせるのは、プロセスのためのプロセスに隠れにくくなるからです。顧客にとって最善のことをし、時宜にかなった具体的なものを約束どおり届けることが問われます。

アジャイル運動は方法論に反対しているのではありません。むしろ、多くの参加者は方法論という言葉の信頼を取り戻したいと考えていました。モデリングは受け入れますが、ほこりをかぶる図を保管庫にしまうためではありません。文書も受け入れますが、保守されず、ほとんど使われない大量の文書のためではありません。計画も立てますが、変化の激しい環境では、**計画には限界がある**ことを認めます。

Snowbird の会合は、2000年春に Oregon の Rogue River Lodge で開かれた、Extreme Programming の支持者と数人の外部参加者による集まりがきっかけでした。その時点では、「Light」あるいは「Lightweight」な方法論という呼び名が使われていましたが、多くの人はその呼び名を気に入っていませんでした。

最終的には Snowbird が開催地となり、そこで「agile」という言葉が選ばれました。この会合から生まれた宣言は、ソフトウェア開発、方法論、組織をより俊敏な方法で考えるための出発点になりました。
