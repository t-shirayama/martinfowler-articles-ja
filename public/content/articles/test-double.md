# Test Double

## 要約

Test Double は、テストのために本番用オブジェクトを置き換えるもの全般を指す総称です。スタントダブルのように、実物の代わりにテストの場に立つオブジェクトだと考えるとわかりやすいでしょう。

この語は、dummy、fake、stub、spy、mock といった似た言葉が混在していた状況を整理するために使われます。それぞれの違いを理解すると、テストで何を置き換えているのか、そしてその置き換えにどんな意図があるのかを明確にできます。

## 読むときの観点

- Test Double は個別の技法名ではなく、置き換え用オブジェクト全体を指す総称として読む。
- dummy、fake、stub、spy、mock は、何をどこまで振る舞わせるかで区別する。
- mock は単に戻り値を返すだけでなく、期待される呼び出しを検証する点に注目する。

## 原文の翻訳

Gerard Meszaros は、さまざまな `xUnit` フレームワークを使うためのパターンをまとめる本に取り組んでいます。その中で厄介なことのひとつが、テストのためにシステムの一部を差し替えるものに対して、stub、mock、fake、dummy など、さまざまな名前が使われていることでした。これに対処するために、彼は独自の語彙を整理しました。私はそれをもっと広める価値があると思っています。

彼が使う総称は `Test Double` です。スタントダブルを思い浮かべるとよいでしょう。**Test Double は、テスト目的で本番用オブジェクトを置き換えるあらゆる場合を指す総称**です。Gerard は、いくつかの種類の double を挙げています。

- `Dummy` オブジェクトは受け渡しされますが、実際には使われません。たいていは、引数リストを埋めるためだけに使われます。
- `Fake` オブジェクトは実際に動く実装を持っていますが、通常は何らかの近道をしているため、本番には適しません。`InMemoryTestDatabase` はよい例です。
- `Stub` は、テスト中に行われる呼び出しに対して、あらかじめ用意された答えを返します。通常、そのテスト用にプログラムされたもの以外には反応しません。
- `Spy` は、呼び出され方に基づいて何らかの情報も記録する stub です。たとえば、送信されたメッセージ数を記録するメールサービスがその一形態です。
- `Mock` は、受け取ることを期待されている呼び出しの仕様として、期待値を事前に設定されたものです。期待していない呼び出しを受け取った場合には例外を投げることができ、検証時には期待していた呼び出しをすべて受け取ったかを確認します。

mock、double などの使い方については、`Mocks Aren't Stubs` でさらに詳しく説明しています。
