# From XP to TCR & Limbo

## 要約

Kent Beck と Daniel Terhorst-North による GOTO Unscripted の対話です。公開されている説明では、Extreme Programming (XP) の進化と、その後のアジャイル開発への影響が主題になっています。

中心にあるのは、短いフィードバックループ、反復的な学習、そしてプロダクトの成熟段階に応じてやり方を変えることです。Beck の 3X モデル、すなわち Exploration、Expansion、Extraction の考え方も、この文脈で扱われています。

また、Test && Commit || Revert (TCR) と Limbo という実験的な手法が紹介されています。どちらも、作業を小さくし、失敗からすばやく学び、**安全な短い反復**を通じて開発を進める姿勢を強調しています。

## 読むときの観点

- XP を過去の手法としてではなく、フィードバックループを短くするための継続的な実験として読む。
- 3X モデルを、プロダクトの段階ごとに最適な開発行動が変わるという視点で捉える。
- TCR と Limbo を、単なるツールやルールではなく、作業単位を小さくするための制約として読む。
- 心理的安全性が、速い反復と責任ある実験を支える条件として扱われている点に注目する。

## 原文の翻訳

Kent Beck と Daniel Terhorst-North は、Extreme Programming (XP) の歩みと、それがアジャイルなソフトウェア開発に残した影響について振り返ります。話題は、短いフィードバックループ、反復を通じた学習、そしてプロダクト開発の段階に応じて適応することへ広がります。そこでは、Beck の 3X モデルである Exploration、Expansion、Extraction も取り上げられます。

Kent は、Test && Commit || Revert (TCR) と Limbo という急進的な技法を紹介します。これらは従来のコーディングや協働のやり方に問いを投げかけ、**ごく小さく安全な反復**を重ねることを重視します。この対話は、Amy C. Edmondson が述べる高い成果を出すチームの要素である心理的安全性とも結びついています。

Kent は、ソフトウェア開発がリアルタイムに近い形でスケールしていく未来を見ています。その未来では、チームはより速い実験、責任ある行動、継続的な学習を受け入れる必要があります。
