# Function As Object

## 要約

Function As Object は、第一級関数とクロージャを使って、データと振る舞いを束ねたオブジェクトのようなものを作る考え方です。
コンストラクタ関数のローカル変数をクロージャで保持し、外に返す関数群を公開メソッドとして扱います。

この文章は、クラス構文がなくてもオブジェクトの性質を作れることを示しつつ、だからクラス構文が不要だとまでは言い切らない、というバランスを取っています。
カプセル化、公開メソッド、private メソッド、メソッド選択の関係を、JavaScript の例で読み解くと理解しやすくなります。

## 読むときの観点

- クロージャがフィールドの役割をどう担うかを見る。
- 返される hashmap が公開インターフェースになる点に注目する。
- クラス構文なしでカプセル化を実現する方法として読む。
- 「関数でオブジェクトを作れる」と「クラス構文は不要」は別の主張だと切り分ける。

## 原文の翻訳

プログラミングにおけるオブジェクトの根本的な考え方は、データと振る舞いをひとまとめにすることです。これにより、関連する関数群を書くときに共通のデータ文脈を持てます。また、データを操作するためのインターフェースを提供できるため、オブジェクトがそのデータへのアクセスを制御し、派生データを支えたり、不正なデータ変更を防いだりしやすくなります。多くの言語は、オブジェクトの定義として働くクラスを定義するための明示的な構文を備えています。しかし、第一級関数とクロージャを持つ言語であれば、Function As Object パターンを使って、これらの構成要素からオブジェクトを作れます。このパターンはもともと Eugene Wallingford が説明したものです。

JavaScript で function-as-object スタイルを使って作った、単純な person オブジェクトの例を示します。日付処理には、Java の日付と時刻処理のひどい混乱を整理した Joda-Time の移植である `js-joda` を使っています。`joda-js` が日付と時刻の扱いに正気を持ち込む仕事を繰り返してくれているのはうれしいことです。

```javascript
function createPerson(name) {
  let birthday;
  return {
    name: () => name,
    setName: (aString) => name = aString,
    birthday: () => birthday,
    setBirthday: (aLocalDate) => birthday = aLocalDate,
    age: age,
    canTrust: canTrust,
  };
  function age() {
    return birthday.until(clock.today(), ChronoUnit.YEARS);
  }
  function canTrust() {
    return age() <= 30;
  }
}
```

function-as-object の外側の形は関数であり、これはコンストラクタ関数として呼び出されます。呼び出しの結果は、本質的には関数の hashmap であり、それがメソッド選択器として働きます。この map は、関数内の変数の状態をクロージャとして捕捉し、データが1回の関数呼び出しを越えて残るようにします。この結果として返される hashmap は、古典的なオブジェクトのように扱えます。JavaScript の用語ではこれは object と呼ばれますが、それは JavaScript のオブジェクトであって、ここで作ろうとしている古典的なオブジェクトとは違います。混乱を減らすため、ここでは hashmap と呼ぶことにします。

```javascript
const kent = createPerson("kent");
kent.setBirthday(LocalDate.parse("1961-03-31"));
const youngEnoughToTrust = kent.canTrust();
```

function-as-object を古典的なオブジェクト指向の観点から見ると、次のようになります。

- オブジェクトのフィールドは、コンストラクタ関数へのパラメータである `(name)` と、ローカル変数である `(birthday)` によって表されます。
- オブジェクトのメソッドは、コンストラクタ関数の中にネストされた関数です。オブジェクトのメソッドと同じように、それらは互いを自由に呼び出し、ローカルスコープにある変数、つまりフィールドを操作できます。
- コンストラクタ関数の外側にあるものは、その変数へアクセスできません。これにより、**データのカプセル化**が保たれます。
- オブジェクトの公開メソッドは、結果として返される hashmap に含まれる関数です。
- コンストラクタ関数の内側にネストされているが、結果の hashmap に含まれていない関数は private メソッドです。
- 公開メソッドの名前は、コンストラクタ関数内の関数名ではなく、結果の hashmap のキーです。混乱を避けるため、私はキーと関数名を同じにしておくのを好みます。ただし、必要であれば関数に別名を付けられるのは便利です。ES6 では、`age: age,` を `age,` に置き換える省略記法を使って、この重複を取り除けます。

このパターンのよくある別実装は、JavaScript における自然なメソッド選択器である hashmap ではなく、関数をメソッド選択器として返す方法です。関数をメソッド選択器として使うなら、呼び出すメソッドの名前を第1引数に取る関数を返します。その関数本体は、その値に基づいて分岐します。この点については Wallingford の説明に詳しく書かれています。

function-as-object のアプローチは長い間存在しています。私は Lisp で何度も説明されているのを見てきましたし、JavaScript でも広く使われてきました。ES6 以前の JavaScript には、クラスという概念が非常に限られた形でしかありませんでした。このアプローチは、クラスのための専用構文は不要だという主張としてよく使われます。これは、オブジェクト好きが「単一の `call` メソッドを持つクラスを書けるのだから、第一級関数は不要だ」と言うのと同じようなものです。その結果、JavaScript の世界では ES6 の class 構文を使うことに反対する人が多くいます。個人的には、**第一級関数と第一級クラスの両方がある**のが好きですし、ES6 の class 構文を好んでいます。
