# Hierarchic Data Model

## 要約

Hierarchic Data Modelは、データを階層または木構造として整理する考え方です。初期のデータベースやプログラム内データ構造ではよく使われましたが、データが複雑になるにつれて、データベースではRelationalDataModelが、メモリ上の構造ではNetworkDataModelが主流になりました。

一方で、階層の単純さには今でも価値があります。XMLとその周辺技術は、RelationalDataModelにおけるSQLのように、階層データを扱う標準的な手段を提供したため、階層モデルに再び注目が集まったと述べています。

## 読むときの観点

- 階層モデルは理解しやすいが、複雑な関連を表すと破綻しやすい。
- データベースの世界とメモリ上のデータ構造では、支配的なモデルが異なる。
- XMLは、階層データを扱う実用的な標準技術として位置づけられている。
- 単純さが弱点にも強みにもなる、というトレードオフに注目する。

## 原文の翻訳

階層データモデルは、データを階層または木構造の形で整理します。初期のデータベースやプログラミング上のデータ構造では、階層モデルがよく使われていましたが、やがて支持を失っていきました。データベースの世界ではRelationalDataModelが支配的になり、メモリ上のプログラミングではNetworkDataModelが主流になりました。これは、階層が単純な整理手段である一方で、データが複雑になると崩れやすいからです。

とはいえ、階層の単純さは否定しがたいものです。階層モデルは今でも価値を示し続けています。現在の階層データモデルの大きな復活はXMLの台頭にあります。XMLとその関連技術は、RelationalDataModelにおけるSQLに似た形で、**階層データを操作するための標準的な機能**を提供しているからです。
