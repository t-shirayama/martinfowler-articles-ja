# November Inflection

## 要約

November Inflection は、2025年11月が AI 支援プログラミングにとって重要な転換点だった、という短い観察です。Anthropic の Opus 4.5 と OpenAI の GPT-5.2 が登場し、とくに Claude Code のような harness と組み合わさったことで、AI を使った開発の実感が一段変わったと述べています。

Fowler は、モデルが少し良くなっただけに見えても、見えない能力線を越えると急に難しいコーディング課題が開ける、という Simon Willison の見方と、手でコードを書く時代から AI エージェントを指揮する時代へ移ったという Adam Tornhill の経験を引いています。年末休暇に多くの人が新しいモデルを試せたことも、この変化の実感を強めました。

## 読むときの観点

- 「転換点」を、単一モデルの性能発表ではなく、モデルと開発ツールの組み合わせとして見る。
- 能力が連続的に上がっても、利用者の体験では非連続な変化として現れる点に注目する。
- コードを書くことから、AI エージェントに作業を進めさせることへの役割変化を考える。
- 休暇のような時間的余裕が、新しい技術を試す社会的な拡散タイミングになる点を見る。

## 原文の翻訳

2025年11月は、AI 支援プログラミングの発展における重要な地点となりました。その月に Anthropic は Opus 4.5 をリリースし、OpenAI は GPT-5.2 をリリースしました。前者は特に Claude Code という harness と組み合わさることで、AI を使ううえで重要な瞬間を示しました。

Simon Willison は、それを、モデルが段階的に良くなった結果、見えない能力の境界線を越え、突然はるかに難しいコーディング問題が数多く開けるようになる瞬間の一つだと表現しています。

Adam Tornhill は、2025年の終わりごろ、自分は最後に手でコードを書いたのだと述べています。その時点では意識していなかったものの、それはほぼ40年にわたるプログラミングからの移行を示す日でした。それ以降も彼は多くのコードを作り、出荷しています。しかし重要な違いが一つあります。そのコードは手で書いたものではなく、**彼が指示した AI エージェントによって書かれたもの**だったのです。

この変化は、クリスマス休暇によってさらに強まりました。数週間のあいだ、日々の仕事が忙しくて AI を試す時間を取れなかった人たちが、これらの新しいモデルを試す機会を得たからです。
