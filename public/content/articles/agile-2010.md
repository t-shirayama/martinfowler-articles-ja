# Agile2010

## 要約

Agile 2010 カンファレンスに参加した Fowler 氏の散発的な印象をまとめた記事です。参加者数、受賞者、発表数、参加者層、プログラミング寄りの話題の少なさなど、当時の agile コミュニティの空気が短く記録されています。

中心にあるのは、agile には技術的な卓越だけでなく、チームや顧客との協働も欠かせないという見方です。プログラミングを軽視してはいけない一方で、プログラミングだけを切り離した場に閉じ込めるのも望ましくない、と述べています。

## 読むときの観点

- Agile 2010 は、agile が主流化しつつあった時期のカンファレンス記録として読む。
- 「soft skills」への重点と、プログラミング側からの批判の両方を見る。
- XP が技術的卓越と協働を結びつけるものとして位置づけられている点に注目する。
- 技術イベントの構成や品質管理についての Fowler 氏の好みも、文脈として読む。

## 原文の翻訳

先週、私は Orlando で開かれた [Agile 2010](https://agile2010.agilealliance.org/) カンファレンスに参加しました。Agile 20xx は、[XP Universe](https://c2.com/cgi/wiki?XpUniverse) と [Agile Development Conference](https://c2.com/cgi/wiki?AgileDevelopmentConference) にルーツを持つ、米国の主要な agile 系カンファレンスです。私は主要な agile カンファレンスの常連ではありませんが、昨年も参加しました。まとまった説明を試みるのではなく、いくつか散発的な印象を書いておきます。

- 参加者は1400人で、2009年より多く、2008年と同じくらいでした。Great Recession の影響は受けたものの、かなりうまく乗り切ったと言えます。
- [Elizabeth Hendrickson](https://testobsessed.com/) と [Liz Keogh](https://lunivore.com/) が [Gordon Pask award](https://www.paskaward.org/) を受賞したのを見て、とてもうれしく思いました。
- 私の感覚では、参加者の多くは agile について比較的新しい人たちでした。これは2009年の私の印象とも一致します。
- 私が普段ソフトウェアカンファレンスで見るよりも、女性の比率がずっと高くなっていました。
- 講演は非常に多く、私には圧倒されるほどでした。16のステージ、つまりトラックで、214の講演がありました。

Elizabeth Hendrickson、別名 testobsessed。

- Jez Humble と私が行った [Continuous Delivery](https://continuousdelivery.com/) のチュートリアルが好評だったようで、うれしく思いました。現在、このチュートリアルは [QCon](https://qconsf.com/) と [OOP](https://www.sigs-datacom.de/) でも再び行う予定になっています。
- 重点の多くは、チームの協働と soft skills に置かれていました。そのため、プログラミング側の人たちから[批判](https://twitter.com/)もありました。
- 私が参加した講演という非科学的なサンプルでは、発表の質が望ましい水準に達していないものの割合が高いように感じました。
- 来年に向けて [XP Universe](https://blog.coryfoy.com/) を再び作ろうという動きがあるようです。
- 私は、休暇用リゾートよりも、本当の都市で開かれるカンファレンスのほうが好きです。
- 今年は、最初の agile 系カンファレンスから10年にあたります。
- カンファレンスは円滑に進みました。これは、すべての主催者の大変な努力の証しです。特に、Nashville の洪水の後でカンファレンスの場所を移す必要があったことを考えると、なおさらです。ありがとう。

Liz Keogh、別名 lunivore。

プログラミングの話題が少ないという不満について、私は複雑な思いを持っています。確かに、プログラミングはソフトウェア開発で中心的な役割を果たしますし、それを周縁に追いやろうとする試みは、行き止まりとよく相関します。しかしプログラミングは、たくさんの要素を含む、非常に相互接続されたシステムの一部です。そして、こうした相互接続を機能させるには、まさにこの種の「soft and woolly」なスキルが必要になります。

プログラマーには、外へ向かってプログラミング以外の人たちと関わるのではなく、内側を向いて技術的な問題に没頭しがちな傾向があります。私が XP で気に入っている点の一つは、XP がこれを認識していることです。つまり、**技術的卓越と協働を混ぜ合わせている**のです。

私は、プログラミングが agilexx カンファレンスから切り離され、プログラマーが別のイベントへ行くようになることは望みません。互いが混ざり合える場所が必要です。[JAOO](https://jaoo.dk/) と [QCon](https://qconferences.com/) は、こうしたことを非常にうまく行っているように見えます。講演そのものの品質管理もより優れています。ただし、その形式がどこでも使えるわけではありませんし、もっと多くの選択肢があってよいでしょう。

私たちにとって重要な課題は、**不可欠な協働を断ち切らずに、プログラミング側の卓越を追求すること**です。
