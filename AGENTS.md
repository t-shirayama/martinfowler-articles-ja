# AGENTS.md

このリポジトリは、martinfowler.com に掲載された記事の非公式日本語訳・要約を管理するためのものです。
作業するときは、公式プロジェクトと誤認されないこと、原文への敬意を保つことを最優先にしてください。

## 基本方針

- Martin Fowler 氏、martinfowler.com、Thoughtworks の公式翻訳であるような表現をしない。
- 元サイトのデザイン、ロゴ、配色、レイアウトをコピーしない。
- 記事には、原文タイトル、原文URL、著者名、作成日、非公式であることを必ず表示する。
- martinfowler.com の FAQ にある翻訳方針に従い、翻訳記事には必ず原文へのリンクを入れる。
- 原文の意味を変えず、日本語として読みやすい表現にする。
- 不確かな訳語は無理に確定せず、必要なら訳注として明記する。

## Markdown の書き方

- 文章は日本語で書く。
- 見出しは短く、読みやすくする。
- コード、ファイル名、クラス名、メソッド名、英語の専門用語は必要に応じて原語を残す。
- 原文へのリンクは記事ページのメタ情報カードに置く。
- 公式サイトと混同されるような装飾やコピー文は避ける。

## 翻訳本文のハイライト

- `## 原文の翻訳` には、原文に基づく翻訳本文を書く。要約、解説、読書メモはこの見出しに混ぜない。
- 謝辞、重要な改訂、主な改訂、改訂履歴などの付録的なセクションは、翻訳本文から省略してよい。
- 全文翻訳か日本語訳・要約かの区別は、記事ページ右サイドバーの翻訳状態で明記する。
- 要約は必ず `## 要約` に書き、翻訳本文の代わりに要約を置かない。
- `## 原文の翻訳` では、特に重要な設計判断、定義、注意点を `**太字**` でハイライトする。
- ハイライトは1段落に1箇所までを目安にし、文章全体が強調だらけにならないようにする。
- 原文にない意味を足して強調しない。強調するのは、原文の主張や定義を日本語で読み取りやすくするために限る。
- 用語の初出、パターンの定義、トレードオフ、誤解しやすい注意点は優先してハイライトする。
- 長い文全体を太字にせず、核になる語句または短い節だけを太字にする。

## 記事メタ情報

- 記事ページでは、右サイドバーのメタ情報カードに原文タイトル、原文URL、著者名、作成日、翻訳状態、非公式であることを表示する。
- Markdown本文の冒頭には、原文URL、著者名、作成日、翻訳状態のメタ情報を重複して書かない。
- Markdown本文は、`# 原文タイトル`、`## 要約`、`## 読むときの観点`、`## 原文の翻訳` の順を基本にする。

## 作業時の注意

- 記事を追加するときは、`src/data/content.ts` と `public/content/articles/` のMarkdownを揃える。
- タグ別の候補ページと翻訳状態を更新するときは、`PROGRESS.md` を更新する。旧 `tags/` 配下の README は削除済み。
- 公開用のタグ本文を更新するときは、`public/content/tags/` 配下のMarkdownを更新する。
- 全文翻訳を追加するときは、権利と出典表記を確認する。
- 大きな構成変更をする前に、変更案を短く説明する。
- 著作権や出典表記に関わる判断は慎重に扱う。
